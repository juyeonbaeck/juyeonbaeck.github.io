가상화 인프라를 운영하다 보면 "서버가 느려요!"라는 컴플레인을 달고 살게 됩니다. 이때 가장 먼저 확인하는 것이 vCenter의 **성능 모니터링(Performance Monitoring) 차트**입니다.


하지만 차트 옵션을 만지다 보면 아주 이상한 점을 발견하게 됩니다. 과거 데이터를 조회할 때는 **평균(Average)이나 최대(Maximum)** 같은 롤업(Roll-up) 값을 선택할 수 있는데, **실시간(Real-time) 조회에서는 이 옵션 자체가 비활성화되어 있습니다.** 그 이유가 무엇일까요?


오늘은 vCenter가 모니터링 데이터를 수집하고 보관하는 **'데이터 압축(Roll-up) 메커니즘'**에 대해 파헤쳐 보겠습니다.


## 🔴 실시간 (Real-time) 모니터링

- **수집 주기:** 20초 간격 <u>_**(20초마다의 그 값을 찍은! 평균값 아님)**_</u>
- **특징:** 데이터베이스에 영구 저장되기 전의 '날것(Raw)' 데이터입니다.
- **설정 불가 이유:** 데이터가 압축(뭉침)되지 않고 실시간으로 점이 찍히기 때문에, 평균(Average)이나 최대(Max) 같은 **Roll-up 연산 옵션 자체를 선택할 수 없습니다.** (선택할 필요가 없음)

## 🔵 보관/과거 데이터 (Historical Data & Roll-up)


20초마다 수집되는 실시간 데이터를 1년 내내 데이터베이스에 그대로 저장한다면 어떻게 될까요? vCenter의 스토리지는 금방 꽉 차서 터져버릴 것입니다(DB 용량 초과 장애).


이를 막기 위해 vCenter는 시간이 지난 오래된 데이터들을 찰흙처럼 하나로 뭉쳐서 압축하는데, 이 과정을 **롤업(Roll-up)**이라고 부릅니다.

- **수집 주기:** 5분, 30분, 2시간 단위 등 (시간이 지날수록 간격이 커짐)
- **특징 (Roll-up):** vCenter DB의 용량 폭발을 막기 위해, 오래된 20초 단위 데이터 여러 개를 하나로 뭉쳐서(Roll-up) 저장합니다.
- **Roll-up 옵션 선택 (필수):** 여러 데이터를 하나로 뭉칠 때 '어떤 대표값'을 남길지 엔지니어가 선택해야 합니다.
    - `Average (평균)`: 전반적인 서버의 부하 추이를 볼 때 사용
    - `Maximum (최대값)`: 특정 시간에 CPU 피크(장애)가 쳤는지 분석할 때 사용 (숨겨진 장애를 찾을 때 강력함)

### 💡 데이터가 압축(Roll-up)되는 과정


시간이 지날수록 데이터의 간격은 점점 듬성듬성해집니다.

    - **최근 1일 (Past Day):** 20초짜리 데이터 15개를 뭉쳐서 → **'5분짜리 데이터 1개'**로 압축
    - **최근 1주 (Past Week):** 5분짜리 데이터 6개를 뭉쳐서 → **'30분짜리 데이터 1개'**로 압축
    - **최근 1달 (Past Month):** 30분짜리 데이터 4개를 뭉쳐서 → **'2시간짜리 데이터 1개'**로 압축

## 3. 실무자를 위한 꿀팁: Roll-up Type 선택하기 🎯


과거 데이터(Past Day, Past Week 등)를 조회할 때 비로소 **Roll-up Type (통계 유형)** 옵션이 활성화됩니다. 15개의 자잘한 데이터를 1개의 대표 데이터로 뭉칠 때, **"어떤 기준으로 뭉쳐서 보여줄까?"**를 엔지니어가 직접 선택해야 하기 때문입니다.


**① Average (평균값)**

    - **특징:** 15개 데이터의 '평균치'를 계산하여 대표로 남깁니다.
    - **언제 쓸까?:** 가장 일반적인 옵션입니다. 서버의 전반적인 부하 추이(Trend)나 평소 리소스 사용량을 파악할 때 사용합니다.

**② Maximum (최대값) - ★ 트러블슈팅의 핵심!**

    - **특징:** 15개 데이터 중 CPU나 RAM이 **'가장 높게 튀었던 순간(Peak)'**의 값을 대표로 남깁니다.
    - **언제 쓸까?:** Average로 보면 평균값의 함정에 빠져 찰나의 순간에 발생한 장애(Spike)를 놓치기 쉽습니다. "어제 새벽 3시에 서버가 10초 동안 뻗었어요!" 같은 미세한 장애 원인을 추적할 때는 **반드시 Roll-up을 Maximum으로 놓고 차트를 분석해야 합니다.**

---


## 🚀 요약 결론

- **실시간(Real-time) 차트:** 데이터가 압축되지 않은 날것(Raw)이므로 Average 선택 불가.
- **과거(Historical) 차트:** DB 용량을 아끼기 위해 데이터를 뭉치며(Roll-up), 이때 뭉치는 기준(Average, Max 등)을 반드시 선택해야 함.
- **장애 분석 시:** 통계의 함정을 피하기 위해 Average보다는 **Maximum(최대값)**을 적극 활용할 것!
