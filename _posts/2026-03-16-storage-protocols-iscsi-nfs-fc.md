vSphere 환경이나 인프라를 구축하다 보면 가장 먼저 마주치는 고민이 바로 **"데이터를 어디에, 어떻게 저장할 것인가?"**입니다.


이들을 **데이터 전송 방식(Block vs File)**에 따라 분류하여 비교해보겠습니다.


---


인프라를 설계할 때 스토리지는 크게 **블록 스토리지(Block Storage)**와 **파일 스토리지(File Storage)**로 나뉩니다. 각 프로토콜의 특징을 이해하면 상황에 맞는 최적의 설계를 할 수 있습니다.


![image.png](/assets/img/posts/storage-protocols-iscsi-nfs-fc-1773712097602.png)


# 1. 블록 스토리지 (Block Storage): "내 컴퓨터의 하드디스크처럼"


데이터를 일정한 크기의 블록(Block) 단위로 관리합니다. OS 입장에서는 마치 내 컴퓨터에 직접 꽂힌 하드디스크(LUN)처럼 인식됩니다.

- **iSCSI (Internet Small Computer System Interface):**
    - **특징:** 일반적인 이더넷(Ethernet) 망을 통해 SCSI 명령을 전달합니다.
    - **장점:** 별도의 전용 장비 없이 기존 네트워크 스위치를 사용할 수 있어 가성비가 매우 좋습니다.
    - **단점:** TCP/IP 오버헤드가 발생하여 성능이 네트워크 상태에 의존적입니다.
- **FC (Fibre Channel):**
    - **특징:** 전용 광케이블과 FC 스위치를 사용하는 인프라의 "끝판왕"입니다.
    - **장점:** 매우 빠르고 안정적이며 지연 시간(Latency)이 거의 없습니다.
    - **단점:** 장비가 매우 비싸고 구축 및 유지보수가 어렵습니다.
- **NVMe-oF (NVMe over Fabrics):**
    - **특징:** 최신 NVMe 기술을 네트워크로 확장한 프로토콜입니다. 차세대 고성능 스토리지의 표준으로 떠오르고 있습니다.

# 2. 파일 스토리지 (File Storage): "공유 폴더처럼"


데이터를 파일과 폴더 구조로 관리합니다. 네트워크를 통해 파일 시스템에 직접 접근하는 방식입니다.

- **NFS (Network File System):**
    - **특징:** 주로 리눅스/유닉스 환경에서 사용하는 네트워크 공유 방식입니다.
    - **장점:** 설정이 매우 간편하고 여러 호스트가 동시에 데이터를 읽고 쓰기에 적합합니다.
    - **단점:** 블록 방식에 비해 오버헤드가 크고 성능이 약간 낮을 수 있습니다.
- **SMB / CIFS (Server Message Block):**
    - **특징:** 윈도우 환경에서 주로 사용하는 공유 폴더 방식입니다.
    - **장점:** 윈도우 서버와의 호환성이 완벽합니다.

# 📊 프로토콜 한눈에 비교하기


| **구분**    | **iSCSI**         | **Fibre Channel (FC)** | **NFS**           | **SMB (CIFS)**    |
| --------- | ----------------- | ---------------------- | ----------------- | ----------------- |
| **인식 방식** | 블록 (Block)        | 블록 (Block)             | 파일 (File)         | 파일 (File)         |
| **전송 매체** | Ethernet (TCP/IP) | Dedicated Fiber        | Ethernet (TCP/IP) | Ethernet (TCP/IP) |
| **비용**    | 저렴함 / 보통          | 매우 비쌈                  | 저렴함               | 저렴함               |
| **복잡도**   | 보통                | 높음 (전용 기술 필요)          | 낮음 (설정 간편)        | 낮음                |
| **주요 용도** | 일반 VM 실행용         | 고성능 DB, 금융권            | ISO, Template 공유  | 윈도우 파일 공유         |


---


### 💡 어떤 것을 선택해야 할까?

1. **가성비와 범용성**을 원한다면? 👉 **iSCSI**
2. **설정의 간편함과 공유 기능**이 중요하다면? 👉 **NFS**
3. **예산이 넉넉하고 최강의 성능**이 필요하다면? 👉 **FC**
4. **차세대 초고속 환경**을 구축한다면? 👉 **NVMe-oF**
